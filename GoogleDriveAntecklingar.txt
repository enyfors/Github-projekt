Förbättringsförslag, process:            
1.       Tags/area, choose (ex. personnel area, work hours, work processes, customer service…)
2.       Headline, write
3.       Problem, write
4.       Solution, write
5.       Do you want to send this message Yes, send it/No, abort
6.       If accepted, time plan and key person(s) responsible for implementation
 Who and how many have to approve solution before implementation/disapprove before rejection.
Number of accepted/rejected solutions
 
Kvittering och kontroll av arbetsuppgifter, process
Work task, area and deadline, choose and/or write
If necessary, repeat this work task every day/week/month, at “00:00” or when X happens.
If necessary, ask for controller to check work task after completion, yes/no
Work task completed at “time stamp”/not completed by worker X
Work task checked to be completed /not completed by controller Y at “time stamp”
 
Arbetsuppgifter och ansvarsområden på individnivå, process
Employee information, choose and write
Employee title and central work tasks: choose
Employee responsibilities, personal and organizational tailored work tasks: choose and write
Employee pay level, additional provision pay: choose and write
Employee work hour/day, work days/week: choose
Option: copy previous employee work profile for recruitment and when new employee is employed
(work tasks and responsible are visible to all)

Användaren skriver in egna arbetsuppgifter och kategorier (inga färdiga etiketter används)
Användare skriver en öppen arbetsuppgift åt teamet, arbetaren kan göra arbetsuppgiften och kvittera att hen gjort den (prestationsbaserad uppföljning, arbetsuppgifternas fördelning)

Förbättringsförslag, hur bestäms och uppföljs de?

Mjukvarans fokus: Planering, utdelning och kvittering av arbetsuppgifter, med fokus på arbetets praktiska genomförande (ej känsliga uppgifter om arbetstagaren, för att minska lagliga problem, läckor eller känsla av spionage, får kartläggas i annan mjukvara)
Arbetsgivaren har rätt att se allas arbetsprestationer, och arbetstagaren har rätt att få feedback framförd och genomförda. Arbetstagaren ser endast sina egna kvitterade arbetsprestationer, ej kollegornas.

Mjukvaran ska kunna användas inom
·       En arbetsplats hela eller delar av personalen emellan (ex. industri, hälsovård, tjänster)
·       Flera arbetsplatser och underleverantörer emellan (ex. husbygge, kultur)
·       Privatpersoner emellan (ex graduskrivande, hushållsuppgifter)

Användnings- och betalningsmodell, förslag:
·       Modell 1: Användare gör gratis personlig profil och kan gratis ta emot arbetsuppgifter från flera huvudanvändare, som mot betalning kan skapa nya projekt (modell typ Google drive)
·       Modell 2: Användare loggar in med projektspecifik pinkod på nätet och kan ta emot arbetsuppgifter, och där huvudanvändare betalar månadsavgift för att skapa nya projekt (Kahoot- modell).
·       Modell 3: Användare loggar gratis in med säker arbets-epost, där arbetsgivaren betalar årslicens för att dela och skapa projekt med personalen.
 
Gratis version: ta emot projekt (och skapa nya) offentligt eller göra max X projekt/dela med X vänner per månad.
Privat användare, köpt tjänst: privat att obegränsat skapa och dela projekt med X antal deltagare
Företag, köpt tjänst: privat att obegränsat skapa och dela projekt med X antal deltagare
 
Mjukvarans framgång mäts med antal unika aktiva användare/månad (modell Kahoot).

Grupparbete och arbetsfördelning: Projektets mål definieras, och varje inbjuden medlem tilldelas ansvarsområden, uppgifter och tidsschema (enskild eller upprepande tidsperiod) som uppfyller målen. 


Planering av arbetet: Varje medlem har kalender, där deltagarna kan fylla i planerade arbetsuppgifter och möten, se andras kalender, boka möten och anteckningar, göra arbetslistor och meddela om frånvaro. 

Nya arbetsuppgifter:
Ledaren skapar ny arbetsuppgift och tidsplan
Ledaren delar ut arbetsuppgifter öppet åt alla (arbetstagaren som påbörjar uppgiften markerar att hen tar ansvaret) eller åt en enskild arbetstagare (kalenderformat)
Arbetstagare kan sätta in förslag på arbetsuppgifter och information som hindrar arbetets genomförande (tex. gräsklippare sönder). 
När uppgiften är färdigt kvitterar arbetstagaren alltid arbetsuppgiften och flera kan kvittera samma arbetsuppgift vid behov. 
Ledaren kan välja om en arbetsuppgift behöver kontrolleras efter kvittering, men arbetstagaren ser inte vilka arbetsuppgifter som kontrolleras. 
Ledaren går igenom arbetstagarnas feedback och röstningar, som används för att göra nya arbetsuppgifter och fördelningar. 

Feedback och förbättringsförslag: medlemmar kan göra ett åtgärdsförslag och om majoriteten/alla av de berörda medlemmarna röstar för förslaget bokförs förslaget som godkänt. 

PROBLEM I DAGENS KONTORSARBETE: för mycket e-post, mycket sökande efter information på intranätet, långa personalmöten utan mål eller praktiska förändringar, ojämlik arbetsfördelning där vissa arbetar för mycket och andra för litet, information gällande arbetet rör sig för långsamt (papperslappar, ringbud, kafferaster)
